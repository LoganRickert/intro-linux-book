Navigating The Filesystem

Navigating the filesystem in Linux can seem frustrating and tedious at first but with a little bit of practice it will become seemingly amazing.

Linux Verses Windows Naming Differences

Before talking about any commands, it's important to breifly touch on the differences between the Linux and Windows file structure. In Windows, it's common to see paths starting with C:\. Windows assigns a different letter to each drive. In figure 1, there are 4 drives, so 4 letters are listed.

Figure 1. My Windows filesystems has a C drive, D drive, E drive and G drive.

If I navigate to G:\npm\introlinux in the explorer, a list of files in that directory are shown. The files can be seen in figure 2. Navigating around the file system in a Linux terminal is very simliar, except that clicking with the mouse isn't available.

Figure 2. The contents of G:\npm\introlinux on my Windows computer.

In Linux, everything starts with a / and doesn't have a concept of 'drives'. My home directory on Linux is: /home/logan. In contrast, my home directory on Windows is: C:\Users\Logan.

It's important to point out two important vocabular terms that you probably know but have never really thought about: File and Folder. A File contains information and a Folder contains a list of filenames and folders. If you look back at Figure 2, the listing magicbook.json is really just a pointer to a file associated with the name "magicbook.json" that contains some data. That data could be anything, from text to an application.
