Chapter 2 Navigating The Filesystem

Navigating the filesystem in Linux can seem frustrating and tedious at first but with a little bit of practice it will become seemingly amazing.

Linux Verses Windows Naming Differences

Before talking about any commands, it's important to breifly touch on the differences between the Linux and Windows file structure. In Windows, it's common to see paths starting with C:\. Windows assigns a different letter to each drive. In figure 1, there are 4 drives, so 4 letters are listed.

Figure 1. My Windows filesystems has a C drive, D drive, E drive and G drive.

If I navigate to G:\npm\introlinux in the explorer, a list of files in that directory are shown. The files can be seen in figure 2. Navigating around the file system in a Linux terminal is very simliar, except that clicking with the mouse isn't available.

Figure 2. The contents of G:\npm\introlinux on my Windows computer.

In Linux, everything starts with a / and doesn't have a concept of 'drives'. My home directory on Linux is: /home/logan. In contrast, my home directory on Windows is: C:\Users\Logan.

It's important to point out two important vocabular terms that you probably know but have never really thought about: File and Folder. A File contains information and a Folder contains a list of filenames and folders. If you look back at Figure 2, the listing magicbook.json is really just a pointer to a file associated with the name "magicbook.json" that contains some data. That data could be anything, from text to an application.

Listing Files In Linux

When a new terminal is opened, it defaults to the /home/yourusername directory. My username on Linux is logan, so my path would look like the following: /home/logan. To list the files in the current directory, the following command can be ran: ls.

Manual #2.1
LS
Command
Description

List information about the FILEs (the current directory by default).

Common Flags
  • l - List extended information about the file
  • a - List all files, including hidden files (files starting with a .
  • h - Show the size of the files in a human readable format.
  • t - Sort by time last modified with the newest file being first.

The output from the command is shown in the top part of figure 3. Each item in the list is either a file or a directory. It can be hard to tell which is a file and which is a directory. By adding the -l flag, as seen in the middle part of figure 3, the command will print out information about each entry. The d at the start of the listings indicate whether or not the entry is a file or a folder. In this case, layouts and stylesheets are both folders, while magicbook.json is a file. Often you'll want to list the contents of a directory without having to change your default directory. You can do this by adding the name of the directory after the flag (if one is present).

logan:~$ ls
build   fonts   includes  layouts   stylesheets
content   images    javascripts magicbook.json
logan:~$ ls -l
drwxr-xr-x  4 logan  logan  128 Mar 12 16:21 layouts
-rw-r--r--  1 logan  logan  569 Mar 12 16:21 magicbook.json
drwxr-xr-x  5 logan  logan  160 Mar 12 16:21 stylesheets
logan:~$ ls -l layouts/
total 16
-rw-r--r--  1 logan  logan  290 Mar 12 16:21 html.html
-rw-r--r--  1 logan  logan  213 Mar 12 16:21 pdf.html
Figure 3. Example output from ls command.
