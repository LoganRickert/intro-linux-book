Chapter 1 Introduction To Linux

Linux is a free, open-source operating system. The design of the operating system is based on Unix, an older operating system. A lot of the work that happens on Linux is done from inside of a text-based application called a terminal. It can be challenging to get use to and work with the terminal at first but with a little bit of practice it becomes blazing fast compared to most graphical user interfaces. It's important to be familiar with the terminal because almost all server-based applications and systems do not have a graphical user interface. Files have to be moved around, applications have to be installed, and much more all have to happen from within the terminal.

Manual

The most important command in this book is the man command. Man is short for "manual" and can be used to display information about almost any command. The syntax for man is man <command name>. An example would be man ls. Throughout this book are quick reference sections for some of the commands, the one for man can be seen below.

Manual #1.1
MAN
Command
Description

Formats and displays the on-line manual pages.

One of the nicest features about man is that it displays the arguments and flags for a command with a description for each. An argument is extra information you pass to a command, such as the name of a file. A flag is an option that you want to enable or disable with the command, such as reversing the order of the output.

Man pages can sometimes be very long. In order to keep them simple to use, it employs the use of the less command automatically. When you run the command, only a small part of the entire page is shown. The up and down arrow keys can be used to scroll up and down and q can be used to stop viewing the man page. You can also highlight and search through the man page by typing a /, typing your search term, and pressing enter.

Man pages can also be viewed online. Just search the command you would have entered and the man page will come up.
