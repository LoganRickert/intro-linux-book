Chapter 5 Networking Tools

We will deal with the lower layers of the networking stack. The transport layer, network layer, data link layer, and physical layer are all needed in order to use the internet. The Transport Layer splits up longer messages into multiple, smaller messages. The Network Layer contains information about where the traffic should go. The Data Link Layer is the infrustructure to figure out how to handle the traffic that is coming in and out. The Physical Layer are the physical connections, such as ethernet cables.

nmap

Manual #5.1
NMAP
Command
Description

Scans devices connected to a network.

Common Flags
  • -p X-Y - specifys to scan from port X to Y
  • -sV - tells you what services are running on the system, takes longer
  • -A - quick good default scan of 1000 most common ports
  • -sn - no port scan, faster and only host discovery

Nmap is used to map out a network and get information about the machines on it. Find out who is on a subnet for example runnmap -sn 192.168.1.1-255. To quickly scan a single machine run runnmap -A 192.168.1.1. To figure out what services might be running run runnmap -sV -p 0-1500 192.168.1.1.

ssh

Manual #5.2
SSH
Command
Description

Establish a secure remote connection to a server.

Common Flags
  • -i [fiilename] - Specifiy the use of an identity file or ssh key
  • -l [username] - Specifiy the username to login with.
  • -p [port] - Specifiy a port.

SSH is used to access a remote system. often it will just be a username and password, signing in looks like this ssh user@host Often times you will need to use a key to sign in ssh -i key user@host Even if you have a password generating a key can make signing in faster and more secure, this is done with ssh-keygen. this generates a public private key pair. The private key(id_rsa) stays on your system, while the public key(id_rsa.pub) goes on the remote system, this is done with ssh-copy-id -i ~/.ssh/id_rsa.pub user@host.

iptables

Manual #5.5
IPTABLES
Command
Description

used to set basic firewall rules, like redirecting or droping network traffic.

Common Flags
  • -s [ipaddress] - specify rule for ipaddress
  • -p - specify protocol, usually tcp, sometimes udp
  • --destination-port -specifiy a port
  • -j [ACTION] - specify what to do with this traffic, usally DROP or ACCEPT
  • -t [table] - specify which table to add the rules to
  • -A - (A)ppend the rule to the current rules on the table

IPTABLES is used to manage network traffic. For example to DROP all INPUT traffic on port 1234 use thisiptables -A INPUT -p tcp --destination-port 1234 -j DROP To specify an ipaddress, like 127.0.0.1 use -s: iptables -A INPUT -p tcp --destination-port 1234 -s 12.0.0.1 -j DROP Sometimes you may want to just redirect traffic, this can be done with this iptables -t nat -A OUTPUT -p tcp --dport [port] -j DNAT --to-destination [ipaddress]:[port] To save current iptables rules use iptables-save > iptables-config which will make a file names iptables-config that has all of your iptables rules, to load a config use iptables-restore < iptables-config
